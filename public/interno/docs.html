03 Login & sessione

Accesso su account/login.php. Supporta username o email, rate limiting, remember-me e redirect post-login.

Autenticazione

Il form accetta sia username sia indirizzo email. Se il campo contiene @ viene tentata la ricerca per email (solo se univoca). Rate limiting: 5 tentativi in 15 minuti per IP.

Resta connesso (remember-me)
AspettoDettaglio
Cookiegh_rm — HttpOnly, Secure, SameSite=Lax
Storage DBHash SHA-256 del token (raw token solo nel cookie)
Scadenza30 giorni, rolling (si rinnova ad ogni accesso automatico)
Per-dispositivoOgni tablet/telefono ha il proprio token indipendente
LogoutCancella solo il token del dispositivo corrente
⚠️

Per installazioni esistenti: CREATE TABLE IF NOT EXISTS remember_tokens (...) — vedi suite/install/schema.sql.

Redirect post-login

Se un utente non autenticato accede a una pagina protetta (es. link QR), require_login() salva l'URL in $_SESSION['login_next'] e lo usa dopo il login invece della dashboard.

White label nel login
  • Suite con logo: mostra il logo caricato in Impostazioni → Identità
  • Suite senza logo: mostra iniziali in gradiente brand accent
  • Pro/Essenziale: mostra il logo GestHall SVG
04 Guida operatore

Cosa può fare un operatore: compilare la cassa, gestire i turni, segnalare guasti, registrare prestiti.

Cassa giornaliera

Pagina principale: cassa/giornaliero.php. Struttura a tab per turno (fino a 3). Swipe touch tra i turni su mobile. Banner colorato per scostamento: verde <4€, giallo 4–5€, rosso >5€.

Contanti per taglio Scassettamenti VLT Ticket pagati Refill AWP Fondo cassa / Monete / Bancomat Note turno 
cassetto   = contanti + refill_awp + differenze - ii_cassa - rientri
versamento = scassettamenti - bancomat - ticket
scostamento= (cassetto + monete) - fondo_cassa - versamento
QR segnalazione guasto

Ogni macchina ha un QR code fisico stampabile da sala/print_qr.php. Il QR punta a sala/segnala_guasto.php: pagina standalone senza login, accessibile da qualsiasi scanner.

1
Scannerizza il QR
Con la fotocamera del telefono. Si apre automaticamente la pagina segnalazione, nessun login richiesto.
2
Descrivi il problema
Campo obbligatorio. Opzionalmente inserisci il tuo nome.
3
Invia
Ticket salvato nel DB + email automatica all'assistenza configurata in Impostazioni.

06 Guida revisore

Cosa il revisore NON può fare

Nome, email, password, revenue share % (es. 30).

13 Super admin

Strumenti per la gestione remota delle installazioni: ghost login diretto, accesso DB, backup.

Ghost login diretto (senza hub)

Vai su https://[url]/account/ghost_generate.php e inserisci la superadmin_key. Il link firmato è valido 15 minuti.

🔒

La superadmin_key è in impostazioni nel DB. Conservarla nel password manager con etichetta chiara per ogni sala. Non condividere i link ghost.

Cambiare piano manualmente
mysql -u gh_sala -p gh_sala
UPDATE impostazioni SET valore='suite' WHERE chiave='piano';
-- valori: 'essenziale', 'pro', 'suite'
Backup DB
# Backup singola installazione
mysqldump -u root -p gh_nomesala > /backup/gh_nomesala_$(date +%Y%m%d).sql

# Backup tutti i DB GestHall
for db in $(mysql -u root -p -e "SHOW DATABASES LIKE 'gh_%'" -s --skip-column-names); do
  mysqldump -u root -p $db > /backup/${db}_$(date +%Y%m%d).sql
done

14 Deploy su SiteGround

Come configurare N installazioni su sottodomini *.gesthallsuite.it con un unico codebase condiviso.

Struttura domini
gesthallsuite.it          → Astro sito marketing (public_html/)
hub.gesthallsuite.it      → Pannello hub PHP (public_html/hub.gesthallsuite.it/)
demo.gesthallsuite.it     → Installazione demo suite
nomesala.gesthallsuite.it → Installazione cliente
Prima installazione suite
1
Clone repository
cd /home/[utente]/ && git clone git@github.com:gesthall-suite/suite.git gestHall-suite
2
Crea sottodominio in SiteGround
Site Tools → Domains → Subdomains → Crea demo.gesthallsuite.it
3
Symlink document root
ln -s /home/[u]/gestHall-suite/suite /home/[u]/www/demo.gesthallsuite.it/public
4
Crea DB MySQL
Site Tools → MySQL: crea DB gh_demo, crea utente, assegna privilegi ALL.
5
Isolamento config via .htaccess
Crea /home/[u]/www/demo.gesthallsuite.it/.htaccess con SetEnv GH_DB_NAME "gh_demo" ecc. Il config.php condiviso legge da getenv().
6
Wizard setup + SSL
Visita /install/setup.php, completa, rinomina il file. SSL da Site Tools → Security → SSL Manager.
Seconda installazione in poi

Ripeti solo i passi 2, 4 (nuovo DB), 5 (nuovo .htaccess), 6 (wizard). Tempo stimato: 10–15 minuti.

Aggiornare tutte le installazioni
cd /home/[utente]/gestHall-suite
git pull origin main
# Un solo pull aggiorna tutti i vhost contemporaneamente
Deploy sito Astro
cd sito/
npm run build        # genera dist/
rsync -avz dist/ user@server:/home/user/public_html/
# oppure: npm run build e upload manuale via File Manager SiteGround
15 Architettura

Modello C — ibrido subdomain: stesso codebase PHP, DB MySQL separato per cliente, vhost diverso per ogni sottodominio.

ModelloDescrizioneVerdict
A — IsolatoServer+DB separati per clienteNon scala gli aggiornamenti
B — MultitenantUn'app, un DB, colonna sala_idRiscrittura massiva + rischio data leakage
C — Ibrido subdomainStesso codebase, DB separati, vhost diversiScelto
DNS layout
*.gesthallsuite.it      → server principale (wildcard A record)
  roma.gesthallsuite.it   → DB: gh_roma
  napoli.gesthallsuite.it → DB: gh_napoli
hub.gesthallsuite.it    → pannello admin separato
Stack tecnologico
PHP 8+ · PDO MySQL 8 / MariaDB 10.6+ HTML/CSS/JS Vanilla PWA · Web Push VAPID Chart.js (CDN) Astro 7 (marketing)
